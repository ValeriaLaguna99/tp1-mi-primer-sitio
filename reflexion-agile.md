# Reflexión sobre Metodologías Ágiles

a) Scrum vs Kanban:
Scrum se basa en periodos de tiempo fijos llamados Sprints con roles definidos, mientras que Kanban es un flujo continuo sin interrupciones temporales. 
Para un proyecto individual, Kanban me parece más útil porque permite visualizar el progreso de forma flexible sin la presión de cerrar un Sprint. 
Es ideal para gestionar tareas personales según la prioridad del momento.
b) MVP (Producto Mínimo Viable):
Un MVP es la versión más simple de un producto que permite que sea funcional y aporte valor al usuario con el menor esfuerzo de desarrollo.
Si un negocio local me pide un sitio web, el MVP sería una página con su ubicación, horario y contacto. 
No necesita un sistema de reservas complejo al inicio, solo lo esencial para que el cliente los encuentre.
